# Фидента — статический сайт

Репозиторий подготовлен для публикации одностраничного сайта ООО «УК Фидента» на GitHub Pages.

## Что внутри

```text
.
├── index.html        # главная страница сайта для GitHub Pages
├── Index(2).html     # исходный HTML-файл, оставлен без изменений
├── README.md         # описание репозитория и инструкция по публикации
├── .nojekyll         # отключает обработку Jekyll на GitHub Pages
├── .gitignore        # исключения для системных и временных файлов
└── .gitattributes    # настройки обработки файлов Git
```

`index.html` — точная копия исходного файла `Index(2).html`. Содержимое HTML не изменялось. Копия добавлена потому, что GitHub Pages использует `index.html` как стандартный входной файл сайта.

## Как загрузить на GitHub

1. Создайте новый репозиторий на GitHub.
2. Загрузите все файлы из этого архива в корень репозитория.
3. Убедитесь, что файл `index.html` лежит именно в корне репозитория, а не внутри дополнительной папки.
4. Сделайте commit, например: `Initial static site`.

## Как включить GitHub Pages

1. Откройте репозиторий на GitHub.
2. Перейдите в **Settings** → **Pages**.
3. В блоке **Build and deployment** выберите **Deploy from a branch**.
4. В поле **Branch** выберите `main`, а в поле папки — `/ (root)`.
5. Нажмите **Save**.

После публикации GitHub покажет ссылку вида:

```text
https://USERNAME.github.io/REPOSITORY/
```

где `USERNAME` — ваш логин GitHub, а `REPOSITORY` — имя репозитория.

## Важно перед публикацией

Формы на сайте сейчас работают как демонстрационные: при отправке показывается всплывающее сообщение в браузере. Для реального сбора заявок понадобится подключить backend, CRM, почтовый сервис или внешний обработчик форм.

Также перед публичным запуском рекомендуется заменить тестовые контакты, ИНН, адрес и ссылки на юридические документы на реальные данные компании.
